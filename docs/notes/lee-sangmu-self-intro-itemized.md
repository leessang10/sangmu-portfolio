# 이상무 자기소개서 (항목형 버전)

## 1. 지원 동기 및 가치관
저는 "가치 있는 것을 제대로 만들 줄 아는 개발자"가 되는 것을 목표로 일해 왔습니다. 학생 시절에는 이론적 완성도에 집중했다면, 실무 3년을 거치며 실제 사용성, 고객 체감, 운영 안정성, 비즈니스 효과를 우선 판단하는 기준을 갖추게 됐습니다. 기술은 목적이 아니라 수단이며, 결국 중요한 것은 사용자와 회사에 실질적인 가치를 만드는 일이라고 생각합니다.

## 2. 직무 강점
저의 강점은 백엔드 중심의 문제 해결력과 운영 관점을 갖춘 실행력입니다. 프로젝트를 맡으면 요구사항 수집 후 DB/API/운영 영향 범위를 먼저 분석하고, 구현 단계에서는 브랜치 분리, 로컬 검증, 개발서버 테스트, 상용 배포까지 체계적으로 진행합니다. 상용 이슈가 발생하면 원인을 빠르게 좁혀 수정·재배포하고 재발 방지 관점으로 개선합니다.

또한 팀 생산성 향상에 집중해 왔습니다. 선택형 자동 배포 스크립트, API 응답/에러 규격 표준화, 프로젝트 구조 정리, 환경변수(.env) 체계 도입 등 반복 비용을 줄이는 개선을 꾸준히 수행했습니다.

## 3. 핵심 경험 및 성과
가장 대표적인 경험은 회사 표준 백엔드 프레임워크를 Express.js에서 NestJS로 전환한 일입니다. 레거시에는 복붙 누적, 미사용 기능 잔존, 하드코딩, 평문 키 노출, 네이밍 불일치 등 유지보수 리스크가 컸습니다. 저는 전환을 주도하고 핵심 설계·구현을 책임하며, 레거시 보일러플레이트를 의도 단위로 분석한 뒤 개선된 NestJS 보일러플레이트로 점진 이관했습니다.

전환은 점진 이관과 신규 프로젝트 선적용을 병행했고, 안정성 검증 후 기존 서비스(잡스)까지 확산했습니다. 그 결과 코드 구조와 규칙이 정착됐고, 유사 API 재사용이 쉬워져 개발 속도와 협업 효율, 신규 인력 온보딩 속도까지 개선됐습니다.

## 4. 협업 및 커뮤니케이션 방식
저는 의견보다 사실 중심, 장황함보다 실행 중심 커뮤니케이션을 지향합니다. 메시지를 보낼 때는 "누가, 무엇을, 언제까지"가 바로 보이도록 정리해 상대가 즉시 행동할 수 있게 합니다. 갈등 상황에서도 감정 대응보다 근거와 맥락으로 조율하며, 제안할 때는 "~가 좋을 것 같습니다" 같은 협업형 표현으로 합의를 이끌어 왔습니다.

## 5. 입사 후 포부
입사 후에는 백엔드 아키텍처 품질과 운영 안정성을 높이는 동시에, AI 도구를 활용한 빠른 검증·구현 체계를 팀에 기여하고 싶습니다. 기능을 빠르게 만드는 것에 머무르지 않고, 사용자 가치와 비즈니스 성과로 이어지는 결과를 꾸준히 만들겠습니다. 장기적으로는 기술과 실행을 연결해 "가치 있는 것을 제대로 만드는 개발자"로 성장하겠습니다.
