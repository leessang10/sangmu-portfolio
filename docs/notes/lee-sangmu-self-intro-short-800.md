# 이상무 자기소개서 (짧은 버전, 약 800자)

저는 "가치 있는 것을 제대로 만드는 개발자"를 목표로 성장해 온 3년차 백엔드 개발자 이상무입니다. 실무를 하며 기술 자체보다 실제 효과를 기준으로 판단하는 습관을 갖게 되었습니다. 요구사항을 받으면 먼저 DB/API/운영 영향과 일정 리스크를 점검하고, 브랜치 기반 개발-로컬 검증-개발서버 테스트-상용 배포까지 책임 있게 수행해 왔습니다.  

특히 팀 생산성을 높이는 개선을 꾸준히 해왔습니다. 서버/브랜치 선택형 자동 배포 스크립트를 만들어 배포 진입장벽을 낮췄고, API 경로/응답/에러 형식 표준화로 협업 예측 가능성을 높였습니다. 하드코딩 설정을 .env로 전환하고, 헬스체크와 graceful shutdown 같은 운영 대비 요소도 선제 반영했습니다.  

또한 회사 표준 백엔드를 Express.js에서 NestJS로 전환하는 과정에서 전환을 주도하고 핵심 설계·구현을 책임했습니다. 레거시 구조를 분석해 점진 이관과 신규 선적용을 병행했고, 안정성 검증 후 기존 서비스(잡스)까지 확산했습니다. 그 결과 구조 일관성, 개발 재사용성, 온보딩 속도, 운영 안정성이 함께 개선됐습니다.  

또한 커뮤니케이션에서도 실행 중심 원칙을 지켜 왔습니다. 장황한 설명보다 "누가, 무엇을, 언제까지"가 보이도록 정리하고, 의견 충돌 시에도 감정보다 근거 중심으로 조율해 팀의 의사결정 속도를 높이려 노력합니다.  

앞으로도 저는 AI를 생산성 도구로 적극 활용하면서, 빠른 구현과 운영 가능한 품질의 균형을 지키는 개발자로 기여하겠습니다. 기술 자체를 과시하기보다 사용자와 비즈니스에 실제로 도움이 되는 결과를 지속적으로 만들겠습니다.
